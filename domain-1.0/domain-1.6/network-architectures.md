

<p><h1>Three-tier architecture</h1></p>

<p><h3>The Core</h3></p>

<p>This is the center of the network where web servers, databases, and applications live.<br>
Many people need access to this tier.</p>

<p><h3>Distribution</h3></p>

<p>This is the midpoint between the core and users, communication between access switches happens here,<br>
as well as managing the path to the end user.</p>

<p><h3>Access</h3></p>

<p>Where the users connect, workstations, laptops, printers, ect.</p>

<p>Three-tier architectures can be connected.</p>

<p><h1>Collapsed Core</h1></p>

<p>This is a two-tier model, great for smaller organizations, it combines the core and distribution layers.<br>
This makes it easier to design, less expensive to implement, but not as resilient.</p>

<p><h1>Traffic Flows</h1></p>

<p>How data moves within a data center.</p>

<p><h3>East-West</h3></p>

<p>Traffic between devices in the same network, relatively fast response times.</p>

<p><h1>North-South</h1></p>

<p>Any traffic leaving the data center, this is the ingress and egress to an outside device.</p>
